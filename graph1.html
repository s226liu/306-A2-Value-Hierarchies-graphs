Value Hierarchy 1

Stakeholder 1: Women stressed about cycle uncertainty
Instrumental Value: Trustworthiness
  • Level 1 — Value
    Trustworthiness
    • Level 2 — Objective
      Communicate predictive uncertainty clearly.
      • Requirement
        Display period predictions as a date range, not a single date.
      • Requirement
        Attach a visible confidence level to every prediction.
      • Requirement
        Show "insufficient data" message when fewer than 3 cycles are logged.
      • Requirement
        Disable ovulation prediction when cycle variance exceeds a defined threshold.
    • Level 2 — Objective
      Distinguish logged data from inferred data.
      • Requirement
        Label entries as "User Logged" or "Estimated."
      • Requirement
        Use distinct visual styling for logged vs predicted data.
      • Requirement
        Separate raw logs and predictions in export files.
      • Requirement
        Include legend explaining data categories in summaries.
    • Level 2 — Objective
      Avoid overstating medical certainty.
      • Requirement
        Include persistent "Not medical advice" notice on insight cards.
      • Requirement
        Prohibit diagnostic language in automated outputs.
      • Requirement
        Provide "Consider consulting a healthcare professional" guidance for flagged symptoms.
      • Requirement
        Require explanation panel for all AI-generated insights.
    • Level 2 — Objective
      Make data practices transparent.
      • Requirement
        Provide in-app plain-language privacy summary.
      • Requirement
        Include accessible "Download my data" function.
      • Requirement
        Include accessible "Delete my data" function.
      • Requirement
        Default optional data sharing to opt-in.
